03 · 模式切换方式
01
Admin 接口
POST /api/admin/set-mode
携带 ?key=ADMIN_KEY 及 body {"mode":"game"}
修改服务端内存变量，立即对所有新打开的用户生效
立即生效
02
修改环境变量
在服务平台将 REVIEW_MODE 设为 true 或 false
需触发重新部署，服务启动时读取一次性初始化
需重启
03
客户端彩蛋
长按 review-page 内"企业破冰"卡片 1 秒
客户端本地切换 reviewMode: false
不通知服务端，仅当前用户生效

06 · WebSocket 关键消息
消息 方向 说明
AUTH C→S 登录鉴权，携带手机号和 token，服务端触发断线恢复检查
JOIN_ROOM C→S 加入 / 重新接管原席位，同一手机号可恢复断线前的位置
ROOM_STATE S→C 全量房间状态广播，客户端整包替换，驱动所有 UI 渲染
GAME_ACTION C→S 游戏操作：选人、投票、任务出牌、验人等，携带 action 类型和参数
CHAT C→S 玩家发言，服务端广播至房间所有成员
PONG 双向 心跳保活，客户端响应服务端 PING，维持连接稳定
07 · AI 系统两层架构
GAME-AI · 实时决策层
server/game-ai.js — 游戏内行为触发，无 LLM 延迟
fillAiPlayers创建 AI 玩家，分配人格、随机名字
autoSpeakIfAiAI 玩家发言（调用 LLM），推进发言队列
autoVoteIfAiAI 玩家投票（调用 LLM）
autoMissionIfAiAI 玩家任务出牌（调用 LLM）
autoProposeIfAiLeaderAI 队长选人（调用 LLM）
autoAssassinateIfAiAI 刺客刺杀（调用 LLM）
triggerAutoplayActions统一托管触发入口，阶段切换后调用
autoplayPropose/Vote/Mission/Lady真人托管：随机决策，1.5s 延迟执行
AI · LLM 决策层
server/ai.js — 调用 DeepSeek API，含记忆与学习
decideSpeak生成发言：含人格、信念状态、情绪、few-shot 例句
decideTeam队长选人：推理哪些席位可信
decideVote投票决策：综合公开信息和私有情报
decideMission任务出牌：好/坏人各有决策逻辑
decideAssassinate刺杀目标：推断谁是梅林
decideRecap生成复盘：每位玩家的行为分析
evaluateGameSpeeches对局结束后评分并存储优质发言
generateAiJournalsAI 玩家对局后写日记，更新记忆

10 · AI 记忆与学习系统
ai_memory
AI 玩家记忆
每场对局结束后 AI 角色写"日记"，下次发言时读取，形成跨局人格一致性
good_speeches
优质发言库
对局结束后 LLM 评分，高分发言存入 SQLite，作为下次 few-shot 示例注入 prompt
strategy_patterns
策略模式库
记录不同角色在不同局面下的决策模式与胜负结果，提升推理准确性
game_logs
对局摘要
每局游戏总结存档，供 AI 复盘和玩家历史统计使用
